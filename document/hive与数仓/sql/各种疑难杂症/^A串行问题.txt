一、背景
数据导入失败，看log日志提示是串行了。想到可能是^A造成的问题。
而^A与\001其实是hive的默认列分隔符。

二、找到问题数据行
因为数据体量大，所以只能根据错误行的日志信息内容，找到关键的几个字段，进行过滤数据，足够到几十行内，可以观察到数据问题
比如限制poiid and 城市id and 组织节点id and 商品id。
从而找到数据问题。

三、临时修复
将这行数据内容过滤掉。

四、彻底解决问题方案
咨询了一下其他同学，发现也没有好的方案可以解决。
有一个提示，但验证了一下，不可行，因为过滤的数据有点多。
id regexp '^[.A-Za-z0-9-]+$' 

五、很奇怪的地方
select id
from 表
where id like '%8F137E5F9AB29A1575%'  是可以查询到错误的数据的。
但是转换成下面两种，却查询不到数据。
##and id like '%^A%'
##and id like '%\001%'
